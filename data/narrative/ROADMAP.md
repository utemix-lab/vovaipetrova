# Roadmap

> Путь от ручного редактирования к автономной системе знаний

---

## Фаза 1: Фундамент ✅

**Статус:** Завершено

- [x] Universe Graph как единый источник истины
- [x] Формат universe.json с 3S контентом
- [x] Визуальный редактор (editor.html)
- [x] dream-graph читает universe.json напрямую
- [x] Godot как опциональный инструмент режиссуры
- [x] Модульная архитектура без жёстких зависимостей

---

## Фаза 2: Наполнение (текущая)

**Цель:** Создать достаточно контента для выявления паттернов

- [ ] Наполнить Universe Graph 20-50 узлами
- [ ] Установить типовые связи между понятиями
- [ ] Найти повторяющиеся паттерны 3S контента
- [ ] Определить "грамматику" сценографии

**Критерий завершения:** Можно описать правила генерации 3S по типу узла и контексту.

---

## Фаза 3: RAG-интеграция

**Цель:** Система понимает свой граф

- [ ] Индексация universe.json для RAG
- [ ] Семантический поиск по графу
- [ ] Ответы на вопросы о структуре системы
- [ ] Автоподсказки при редактировании

**Технологии:** 
- Embeddings узлов и 3S контента
- Векторная БД (локальная или облачная)
- LLM для генерации ответов

---

## Фаза 4: Автогенерация контента

**Цель:** LLM генерирует 3S по паттернам

- [ ] Шаблоны 3S для разных типов узлов
- [ ] LLM дополняет контент по контексту
- [ ] Человек проверяет и корректирует
- [ ] Feedback loop улучшает качество

**Критерий:** 80% 3S контента генерируется автоматически с минимальными правками.

---

## Фаза 5: Автономность

**Цель:** Система развивается с минимальным участием человека

- [ ] Автоматическое обнаружение пробелов в графе
- [ ] Предложения новых узлов и связей
- [ ] Самовалидация связности
- [ ] Версионирование и откат изменений

**Роль Godot:** Полностью убирается из цепочки. Архивируется для будущих экспериментов.

---

## Метрики успеха

### Количественные

| Метрика | Фаза 2 | Фаза 3 | Фаза 4 | Фаза 5 |
|---------|--------|--------|--------|--------|
| Узлов в графе | 50 | 100 | 200 | 500+ |
| % автогенерации | 0% | 20% | 80% | 95% |
| Время добавления узла | 10 мин | 5 мин | 1 мин | авто |

### Качественные

- **Понятность:** Новый человек понимает систему за 5 минут
- **Связность:** Любые два узла связаны путём ≤ 3 рёбер
- **Полезность:** 3S контент отвечает на реальные вопросы

---

## Риски и митигация

### Риск: Переусложнение графа

**Митигация:** Лимит на глубину (10 уровней), периодический аудит "мёртвых" узлов.

### Риск: Низкое качество автогенерации

**Митигация:** Человек всегда в цикле, feedback улучшает промпты.

### Риск: Потеря фокуса

**Митигация:** Каждый узел должен отвечать на вопрос "зачем это здесь?".

---

## Принципы развития

1. **Инкрементальность** — маленькие шаги, частые проверки
2. **Обратимость** — любое изменение можно откатить
3. **Измеримость** — метрики для каждой фазы
4. **Прагматизм** — работающее важнее красивого
