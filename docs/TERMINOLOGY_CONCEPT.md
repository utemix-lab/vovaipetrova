# TERMINOLOGY CONCEPT

**Дата:** 16 февраля 2026  
**Статус:** Концепция

---

## Принцип

```
Система хранит указатели и связи, не энциклопедию.
```

Термин в системе — это **ID**, а не определение. Описание термина живёт во Внешнем мире (Wikipedia, документация, учебники).

## Иерархия каталогов

```
СУЩНОСТИ (записи)              КЛАССИФИКАТОРЫ
─────────────────              ──────────────

vst-plugins.json               vst-categories.json (иерархия)
companies.json                 tags.json (плоский)
countries.json

Serum, Xfer, US                wavetable, ADSR, envelope, LFO
```

### Сущности

Конкретные объекты Внешнего мира:
- Плагин (Serum)
- Компания (Xfer Records)
- Страна (США)

### Классификаторы

**Категории** (`*-categories.json`):
- Иерархические
- Специфичны для домена (vst-categories, ai-categories)
- Пример: `synth → wavetable`

**Теги** (`tags.json`):
- Плоские (без иерархии)
- Кросс-доменные
- Термины, не вошедшие в другие каталоги
- Пример: `ADSR`, `envelope`, `LFO`

## Логика кликабельности

```
Текст обзора: "Serum 2.0 использует [[wavetable]] синтез с [[ADSR]]..."

Парсер находит [[термин]]:

1. Ищет в vst-categories.json
   → найден "wavetable" → кликабелен → страница категории

2. Ищет в tags.json
   → найден "adsr" → кликабелен → страница тега

3. Не найден нигде
   → обычный текст (не кликабелен)
```

## Структура тега

```json
{
  "id": "adsr",
  "name": "ADSR",
  "name_ru": "ADSR",
  "domain": "music",
  "related": ["envelope", "amplitude"]
}
```

### Поля

| Поле | Описание |
|------|----------|
| `id` | Уникальный идентификатор |
| `name` | Название (EN) |
| `name_ru` | Название (RU) |
| `domain` | Домен (music, ai, design, software, media, games, knowledge) |
| `related` | Связанные теги или категории |

### Чего НЕТ

- `definition` — система не хранит определения
- `description` — система не хранит описания
- `hierarchy` — теги плоские

## Принципы

1. **Термин = указатель, не определение**
   - Система знает, что термин существует
   - Система не знает, что он означает

2. **Кликабельно только то, что в каталогах**
   - Нет записи → нет клика
   - Важный термин → добавить в tags.json

3. **Связи важнее описаний**
   - `related` связывает термины между собой
   - Навигация через связи, не через поиск определений

4. **Домен обязателен**
   - Каждый тег принадлежит домену
   - Это единственная "категоризация" тегов
