# Архитектурные принципы Vovaipetrova

## Ключевой акцент: Графоцентричность

Система архитектурно **графоцентрична**. Это означает:

1. **Граф — основа всего.** Онтологический граф является единственным источником истины. Не действия пользователя, не UI-события, а структура и связи узлов графа.

2. **Интерфейс — проекция графа.** UI существует для того, чтобы человек мог путешествовать по графу. Это классический веб-интерфейс с паттерном "клик → реакция". Никакой "сознательной" реакции системы на действия пользователя.

3. **Мост к будущей сложности — не поведение пользователя.** Для RAG, LLM и рефлексии важны:
   - **Онтология узлов** — типы, связи, метаданные
   - **Инструменты** — pointer-tags, режимы навигации
   - **Режимы** — query mode, routes, exports
   
   Не конкретные клики пользователя.

## Текущая фаза: Визуал-first

Мы идём от обратного: **сначала визуал, потом архитектура**.

```
Фаза 1 (СЕЙЧАС): Визуальный каркас
├── Простой интерфейс для путешествия по графу
├── Компактная онтология (не второй Интернет)
├── Pointer-tags для навигации
└── React как мост для будущей сложности

Фаза 2 (ПОТОМ): Онтологический слой
├── Формализация типов узлов
├── Связи между концептами
└── Машиночитаемые метаданные

Фаза 3 (ПОЗЖЕ): Рефлексия
├── RAG на основе графа
├── LLM-интеграция
└── Саморефлексивные механизмы
```

## Что НЕ делаем сейчас

- ❌ Никакой "магии" — система не интерпретирует намерения пользователя
- ❌ Никаких агентов, реагирующих на клики
- ❌ Никакого семантического слоя между UI и бизнес-логикой
- ❌ Никакой LLM-интеграции

## Что делаем сейчас

- ✅ Строим простой, красивый интерфейс
- ✅ Выстраиваем онтологический граф
- ✅ Закладываем якоря прототипирования (`@ArchProto`, event-система)
- ✅ Используем React для сложных анимаций и как мост к будущему

## Почему такой подход

> "Мне понятнее мыслить визуально, а не архитектурно."

Визуальное мышление позволяет:
1. Увидеть, как система будет выглядеть для человека
2. Нащупать правильные паттерны через прототипирование
3. Не уходить в фантасмагорию абстрактной архитектуры

Архитектура проявится из визуала, а не наоборот.

## Критерий перехода к следующей фазе

Переход к Фазе 2 возможен, когда:
- [ ] Визуальный интерфейс полностью функционален
- [ ] Онтологический граф выстроен и стабилен
- [ ] Pointer-tags покрывают основные сценарии навигации
- [ ] Routes и exports работают как задумано

Только тогда начинается работа с RAG и рефлексией.
