# Ремонтная мастерская (Repair Shop)

Папка для фиксации архитектурных диагнозов, выбранных маршрутов исправления и результатов.

## Структура

```
repair-shop/
├── README.md                    # Этот файл
├── 2026-02-12_DIAGNOSIS.md      # Полный диагноз архитектуры
├── ROUTES.md                    # Возможные маршруты исправления
├── DECISIONS.md                 # Принятые решения и их обоснование
└── RESULTS.md                   # Результаты применённых методов
```

## Принцип работы

1. **Диагноз** — фиксируется полностью, со всеми деталями
2. **Маршруты** — все возможные пути, не только выбранный
3. **Решения** — что выбрали и почему
4. **Результаты** — что дал каждый метод

## Зачем это нужно

При движении по одному маршруту теряется контекст развилки.
Эта папка позволяет вернуться к корневым вопросам и проверить:
- Что дал выбранный метод?
- Что было бы при другом выборе?
- Какие симптомы остались?
