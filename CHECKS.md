# Проверки в проекте

Этот документ описывает все проверки, которые выполняются в проекте перед отправкой изменений на GitHub.

## Список проверок

### 1. Тесты
- **Описание**: Автоматические тесты для проверки корректности кода и данных.
- **Расположение**: `tests/`
- **Команда для запуска**: `npm test`

### 2. Линтинг
- **Описание**: Проверка кода на соответствие стилю и стандартам.
- **Расположение**: Конфигурация в `.eslintrc`.
- **Команда для запуска**: `npm run lint`

### 3. Валидация данных
- **Описание**: Проверка JSON-файлов на корректность и соответствие контрактам.
- **Расположение**: `tests/validate-ui-contracts.js`
- **Команда для запуска**: `npm run validate`

### 4. Markdown-линтинг
- **Описание**: Проверка Markdown-файлов на форматирование.
- **Расположение**: `docs/`
- **Команда для запуска**: `npm run lint:md`

## Автоматизация

Перед каждым коммитом и пушем выполняются следующие проверки:
- Линтинг кода.
- Тесты.
- Валидация данных.

## CI/CD

На GitHub настроен автоматический запуск всех проверок при создании pull request.
