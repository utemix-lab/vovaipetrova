# ONTOLOGY SPECIFICATION v1.0

**Track 1 — Universe**
**Дата:** 13 февраля 2026

---

## 1. Принцип

```
Мир — это распределённый граф без центра.
```

Корневые узлы (Universe, Cryptocosm) выполняют **навигационно-визуальную функцию** и не являются онтологическими источниками смысла.

```
Мир не иерархичен.
Мир не древовиден.
Мир — сеть различений.
```

---

## 2. Типы узлов

### 2.1 System Nodes

| Тип | Назначение |
|-----|------------|
| **Universe** | Контейнер видимого графа |
| **Cryptocosm** | Альтернативная сцена (метафорическая, UI-режим) |

Это **рамки отображения**, а не домены знания.

---

### 2.2 Domain Nodes

**Фундаментальные области знания внешнего мира.**

Текущий набор:

| Domain | Описание |
|--------|----------|
| AI & Computational Models | Искусственный интеллект, вычислительные модели |
| Software & Digital Tools | Программное обеспечение, цифровые инструменты |
| Visual & Media Arts | Визуальное искусство, медиа |
| Music & Sound Culture | Музыка, звуковая культура |
| Design & Aesthetic Systems | Дизайн, эстетические системы |
| Knowledge Systems & Cognitive Architectures | Системы знаний, когнитивные архитектуры |
| Interactive Systems & Game Culture | Интерактивные системы, игровая культура |
| Architecture & Physical Realization | Архитектура, физическая реализация |

**Характеристики Domains:**

```
- ограничены
- устойчивы
- не генерируются динамически
```

---

### 2.3 Character Nodes

**Персонаж — субъект фокусировки.**

**Характеристики:**

- асимметричная компетенция
- стиль
- предпочтения
- слепые зоны
- потенциал активности

```
Персонаж может существовать без Workbench,
но без него считается неактивным.
```

---

### 2.4 Workbench Nodes

**Workbench — навигационная линза персонажа.**

Это:

- способ структурирования его взгляда
- фокус на подмножестве домена
- форма организации внимания

```
Workbench не является Operator.
Workbench — сцена применения Operator.
```

**У одного персонажа может быть несколько Workbench.**

---

### 2.5 Collab Nodes

**Collab — устойчивый тип узла.**

Это:

- объединение двух и более персонажей
- пересечение доменов
- проектная сцена

**Collab может иметь:**

- название
- айдентику
- workflow
- рецепты
- стек

```
Количество Collab не фиксировано,
но их появление — редкий и осмысленный акт.
```

## 3. Типы связей (минимальный набор)

| Связь | Описание |
|-------|----------|
| `Character → operates_in → Domain` | Персонаж работает в домене |
| `Character → has → Workbench` | Персонаж владеет воркбенчем |
| `Workbench → focuses_on → Domain` | Воркбенч фокусируется на домене |
| `Character → participates_in → Collab` | Персонаж участвует в коллаборации |
| `Collab → intersects → Domain` | Коллаборация пересекает домен |

```
Связи не утверждают истину.
Они определяют допустимость навигации.
```

## 4. Что Мир НЕ включает

Мир **не включает**:

- Epistemic Operators
- Генеративные механики
- Нарративные слои
- Внешние данные
- Платформенные контракты

```
Мир — это сцена.
```

## 6. Projection Layer (Non-Ontological)

**UI-навигация, подсветки и режимы отображения не являются частью онтологии мира.**

### Типы узлов в UI

Используются для:
- навигации
- подсказок
- инструментов

**Но:**
- не обязаны существовать в `schema.json`
- не обязаны соответствовать онтологическим типам напрямую
- могут быть временными

### Ключевое различие

```
UI-типы не являются онтологическими типами.
Schema не обязана отражать UI-классификацию.
```

### Три уровня

| Уровень | Назначение | Пример |
|---------|------------|--------|
| **Онтология** (Track 1) | Что существует | Character, Domain, Workbench |
| **Schema** (Track 2) | Что хранится | Структуры данных, поля, связи |
| **Projection** (Track 4) | Как отображается | Подсветки, режимы, метафоры |

**Онтология** — уровень мира.
**Schema** — реализация (может быть нормализована, оптимизирована, упрощена).
**Projection** — трансформация отображения.

## 7. Graph Invariance Principle

**Базовый граф мира инвариантен.**

### Инструменты могут:

- трансформировать **проекцию** графа
- активировать временные метафорические связи
- скрывать/раскрывать подмножества

### Инструменты НЕ могут:

- создавать новые сущности онтологии
- переписывать структуру мира
- модифицировать базовые связи

### Формула

```
Graph = invariant
Projection = mutable
```

### Пример

```
Когда мы захотим показать метафорическую связь
Universe и Cryptocosm — это трансформация проекции,
а не самого графа.
```

### Почему это критично

**1. Защита от UI-детерминизма**
Интерфейс не должен определять структуру мира.

**2. Аккуратное введение Operators**
Operators будут работать:
- с графом (редко, осознанно)
- или с проекцией (часто, безопасно)

И это нужно будет различать.

**3. Глубокий режим без перегрузки UI**
Временные метафорические связи активируются при глубоком изучении системы — это projection-активации, не изменения графа.
